BuJo Ritual — Event Flow

Four-lane orchestration: Mike ↔ Hobbes ↔ bujo-scribe ↔ Apple Notes. Phases color-coded.

Phase legend

🗺️ Router
Pure date logic. Runs once per /bujo, queues rituals.
⚙️ Mechanical
Hobbes dispatches → scribe executes → returns verified diff. Main context stays clean.
💬 Interactive
Strictly Mike ↔ Hobbes. Scribe not involved until decisions are collected.

Invariants the flow enforces

  • Scribe always reads 📓 Journal Index on every dispatch — no cached rules.
  • Every Apple Notes mutation is read-fresh → write, contained inside the scribe.
  • Interactive steps never delegate until Mike has responded — no fabricated answers.
  • Scribe returns diffs, not just ok — ambiguous mutations surface to Mike.

Scribe task vocabulary

  • scribe.read(…) — fetch notes, return packet.
  • scribe.scaffold(…) — create or merge a ritual entry.
  • scribe.apply-decisions(…) — apply signifiers / mutations to a note.
  • scribe.scan(…) — find open or due items across notes.
  • scribe.summarize(…) — produce display block from a packet.

Skeleton reuse

Weekly, monthly, and yearly rituals use the same four-lane skeleton — different interactive prompts, different scribe.* payloads, same contract.
